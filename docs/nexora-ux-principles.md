# Nexora UX Principles

## 1. Target User

Type-C / executive / manager users:
- Want clarity fast.
- Prefer low cognitive load.
- Need a quick next step.
- Do not want clutter.
- Trust concise, grounded signals more than dense UI.

## 2. First 5 Seconds Rule

Within 5 seconds, the user should understand:
- What is wrong.
- How serious it is.
- What to do next.

## 3. First 30 Seconds Rule

Within 30 seconds, the user should understand:
- Why it matters.
- What happens if action is taken.
- Whether the system is trustworthy.

## 4. Right Panel UX Rules

- Keep it compact.
- Summary first.
- No heavy execution.
- Max 2-3 important actions.
- Use specific labels.
- Help Panel is tiny and last.
- Prefer one clear next step over a menu of possibilities.

## 5. Top Bar UX Rules

- Minimal.
- No chat input.
- Primary command only.
- Status and small utilities only.
- Do not duplicate right-panel headings or bottom-dock actions.

## 6. Bottom Dock Rules

- Main input lives here.
- Treat it as a command surface, not messenger UI.
- Suggestions must be short.
- Keep prompts operational and context-aware.
- Do not let chat compete with decision UI.

## 7. Scene Rules

- Scene is sacred.
- No floating chatbot over the scene.
- Object click must respond.
- Empty click should not feel broken.
- Scene should communicate system state, not become a form or dashboard.

## 8. Anti-Patterns

Avoid:
- Too many buttons.
- Generic labels.
- Panel flash/hide.
- Chat competing with decision UI.
- Duplicated headers.
- Empty/fallback-heavy user experience.
- Dense tables in the right rail.
- Long explanations before a recommendation.

## 9. Design Principle Summary

Nexora should feel like a decision console, not a dashboard and not a chatbot.
