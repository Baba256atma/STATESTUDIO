## Scope

DS-5:3 classifies objective health from DS-5:2 progress profiles. Classification and persistence only — no UI, dashboard, assistant, risk, or scenario integration.

DS-5:3 does **not** calculate progress. It reads progress profiles generated by DS-5:2.

Runtime path:

```
Objective → Key Results → OKR Progress Profile → OKR Health Engine → OKR Health Profile → Persistence
```

## Ownership Rule

**DS-5:2 owns all OKR progress calculations.**

DS-5:3 reads progress via:

- `getWorkspaceOkrProgressProfile()`
- `getWorkspaceOkrProgressProfiles()`

DS-5:3 **must not** recalculate objective or key result progress or duplicate DS-5:2 logic.

## Health Profile Contract

`WorkspaceOkrHealthProfile` fields:

| Field | Description |
|-------|-------------|
| `workspaceId` | Owning workspace |
| `objectiveId` | Source objective identifier |
| `healthScore` | DS-5:2 score with trend adjustment (0–100) |
| `healthStatus` | `healthy`, `watch`, `warning`, `critical`, `unknown` |
| `severity` | `none`, `low`, `medium`, `high`, `critical` |
| `healthReason` | Deterministic classification reason |
| `progressPercent` | From DS-5:2 progress profile |
| `variance` | From DS-5:2 progress profile |
| `trend` | From DS-5:2 progress profile |
| `calculatedAt` | ISO timestamp |
| `source` | `ds-5:3-okr-health` |

## Classification Rules

### Health status (from `progressPercent`)

| Progress | Status |
|----------|--------|
| ≥ 100 | healthy |
| 90–99 | watch |
| 70–89 | warning |
| < 70 | critical |
| Missing profile | unknown |

### Severity

| Status | Base severity | Escalation |
|--------|---------------|------------|
| healthy | none | — |
| watch | low | — |
| warning | medium | high if declining |
| critical | high | critical if declining |
| unknown | medium | — |

### Health score

Base: DS-5:2 `score`

Trend adjustments: improving +5, stable 0, declining −5, unknown −10 (clamped 0–100)
