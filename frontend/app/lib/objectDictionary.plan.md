# Object Dictionary Expansion Plan (Core 14 → 49)

## Strategy
- Keep the current **Core 14** stable and opinionated (quality-first, cross-domain).
- Expand in waves of 5–7 objects, grouped by distinct themes: capacity/operations, trust/people, market/strategy, conflict/governance, resilience/recovery, discovery/innovation.
- Before adding any new object, check:
  - Does it cover a unique pattern not already in Core 14?
  - Does it have clear domain examples (business, coaching, politics/futures, story/games)?
  - Does it have at least 6–10 English signals and a short Persian meaning (1–2 lines)?
  - Do its links reference only existing objects (no dead references)?

## No-overlap Guidelines
- One meaning per object: avoid near-duplicates (e.g., “capacity limit” vs “bottleneck”).
- If two candidates feel similar, merge them or mark one as an alias.
- Keep obj_37 as the canonical **“Growth Crisis / Scaling Pain”**; do not create a second scaling object.
- Use precise Persian phrasing (۱–۲ خط) to avoid semantic drift.
- Signals must be concrete English keywords (verbs/nouns), not abstract slogans.

## Linking Rules (cause/effect/support/conflict)
- `links` should clarify relationships using these intents:
  - **Cause**: this object often triggers another (e.g., risk → mitigation).
  - **Effect**: this object is a common outcome of another (e.g., bottleneck → delay).
  - **Support**: this object stabilizes or reduces another (e.g., feedback → risk).
  - **Conflict**: this object counteracts or competes with another (e.g., quick fix × learning).
- Only link to objects that already exist in the dictionary; add new links when the referenced object is live.

## Path to 49 (high level)
- Wave 1 (15–21): operations depth (bottleneck variants, coordination, latency, resilience).
- Wave 2 (22–28): trust/people and conflict patterns (alignment, escalation, morale/energy).
- Wave 3 (29–35): market/strategy signals (competition, positioning stress, timing).
- Wave 4 (36–42): resilience/recovery and learning loops (debt, cleanup, redundancy).
- Wave 5 (43–49): discovery/innovation and futures (experimentation variants, optionality, horizon scanning).

Each wave must pass overlap checks and link back into Core 14 to stay coherent.
